# food-delivery-app-redesign-case-study
End-to-end UX case study covering research, competitor analysis, user flows, information architecture, wireframes, high-fidelity designs and prototyping for a food delivery platform.

## Overview

This project showcases an end-to-end UX design process for a food delivery platform focused on delivering home-cooked meals through verified local home chefs.

The objective was to analyze existing food delivery applications, identify usability issues, understand user behavior, and redesign critical user journeys to create a more intuitive, efficient, and user-friendly experience.

---

## Project Goal

To improve the overall user experience by:

- Reducing cognitive load
- Improving visual hierarchy
- Simplifying navigation
- Enhancing food discovery
- Streamlining the checkout process
- Creating a more engaging and accessible interface

---

## Design Process

### Phase 1: Research & Competitive Analysis

Conducted detailed analysis of leading food delivery applications:

- Zomato
- Swiggy
- EatSure
- TIFFIT

Activities Performed:

- Competitor Analysis
- UI Audit
- Heuristic Evaluation
- Benchmarking
- Feature Comparison
- Strength & Weakness Analysis

### Key Findings

- Existing platforms offer extensive features but often suffer from cluttered interfaces.
- Users experience cognitive overload due to excessive banners, offers, and promotional content.
- Checkout flows frequently contain unnecessary distractions.
- Information hierarchy is inconsistent across multiple screens.

---

### Phase 2: UX Strategy

Created and documented:

- User Flows
- Checkout Flow
- Information Architecture
- Moodboard

Objectives:

- Improve navigation efficiency
- Reduce user friction
- Create a structured information architecture
- Establish a consistent user journey
